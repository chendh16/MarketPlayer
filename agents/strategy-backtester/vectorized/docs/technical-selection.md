## 2. 候选库分析

### 2.1 VectorBT

**优点**:
- 专为回测设计，内置信号生成和风险管理
- 支持向量化订单处理
- 性能极佳 (20-100x)

**缺点**:
- 仅支持 Python
- 文档较少，学习曲线陡
- 与现有 Node.js 系统集成困难

**结论**: 不适合当前架构 ❌

### 2.2 TA-Lib

**优点**:
- 100+ 技术指标
- C 实现，性能高
- 广泛使用

**缺点**:
- 安装复杂 (需要编译)
- 不支持向量化批量计算
- 与 NumPy 集成需要包装

**结论**: 可作为指标库备选 ⭐

### 2.3 Pandas-TA

**优点**:
- 纯 Python，易安装
- 40+ 指标
- 与 Pandas 完美集成

**缺点**:
- 性能一般 (未优化)
- 不支持 JIT

**结论**: 可作为轻量选择 ⭐

### 2.4 Numba JIT (推荐)

**优点**:
- 无缝集成 NumPy
- JIT 编译加速 10-50x
- 自动向量化
- 零代码改动

**缺点**:
- 需要 NumPy 数组操作
- 不支持所有 Python 特性

**结论**: 最佳选择 ✅

## 3. 推荐架构

```
┌─────────────────────────────────────────────┐
│           策略层 (Python/Numba)             │
│  - signal_generator.py (指标计算)           │
│  - backtest_engine.py (向量化回测)          │
│  - risk_manager.py (风险管理)              │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│           引擎层 (Numba + NumPy)            │
│  - 指标向量化: @njit 编译                  │
│  - 数据批处理: 一次性计算全历史             │
│  - 订单处理: 向量化持仓管理                │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│           数据层 (PostgreSQL + 文件)        │
│  - K线数据: NumPy 二进制缓存              │
│  - 结果存储: PostgreSQL                    │
│  - 热数据: Redis 缓存                      │
└─────────────────────────────────────────────┘
```
