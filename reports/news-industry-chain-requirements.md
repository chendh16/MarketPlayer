# 新闻影响模块 - 产业链映射需求文档

## 1. 背景

当前新闻影响模块已实现基础功能：
- 事件检测 (9类事件)
- 情感分析 (词典+事件融合)
- 因子构建 (情感/热度/关注度/累积)
- 信号融合

**待解决问题**：
- 新闻与股票的关系是独立的，没有产业链传导
- 不同行业的新闻影响范围不同
- 港股/美股需要不同的产业链图谱

---

## 2. 目标

构建产业链映射模块，实现：
1. 新闻事件 → 关联行业识别
2. 行业 → 产业链股票映射
3. 产业链传导 (下游→中游→上游)
4. 多市场支持 (A股/港股/美股)

---

## 3. 功能需求

### 3.1 行业识别

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| F1.1 | 根据新闻关键词识别关联行业 | P0 |
| F1.2 | 支持一个新闻关联多个行业 | P0 |
| F1.3 | 行业关键词库可配置/扩展 | P1 |

**行业分类**：
```
- 新能源汽车
- 医药生物
- 房地产
- 半导体/芯片
- 消费电子
- 食品饮料
- 银行金融
- 军工
- 基建/建材
- 互联网科技 (港股/美股)
```

### 3.2 产业链映射

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| F2.1 | 建立行业→产业链→股票映射 | P0 |
| F2.2 | 区分上/中/下游及传导系数 | P0 |
| F2.3 | 支持按市场(A股/港股/美股)切换 | P0 |
| F2.4 | 使用沪深300数据做A股股票池 | P1 |

**产业链数据结构**：
```typescript
interface IndustryChain {
  name: string;
  market: 'a' | 'hk' | 'us';
  levels: {
    downstream: { stocks: string[]; coefficient: number };  // 下游=1.0
    midstream: { stocks: string[]; coefficient: number };    // 中游=0.8
    upstream: { stocks: string[]; coefficient: number };     // 上游=0.6
  };
}
```

### 3.3 传导计算

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| F3.1 | 事件情感 × 传导系数 = 影响得分 | P0 |
| F3.2 | 累积新闻影响 (多日) | P1 |
| F3.3 | 产业链联动 (跨行业传导) | P2 |

### 3.4 多市场支持

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| F4.1 | A股产业链图谱 (50+股票) | P0 |
| F4.2 | 港股产业链图谱 (30+股票) | P1 |
| F4.3 | 美股产业链图谱 (30+股票) | P1 |
| F4.4 | 自动识别市场代码 | P0 |

## 4. 非功能需求

## 5. 技术方案

```
├── industry-chain.ts      # ⭐ 新增: 产业链映射

```

```
   ↓

   ↓
```
