# MarketPlayer 信息收集规则

## 一、定期收集任务

### 1. 竞品分析 (每天)
- **时间**: 7:00 UTC+8
- **内容**:
  - GitHub AI交易助手竞品
  - 新功能特性分析
  - 生成报告
- **输出**: `reports/竞品分析报告_YYYYMMDD.md`

### 2. 市场简报 (每天)
- **时间**: 9:00 UTC+8
- **内容**:
  - A股/港股/美股行情
  - 新闻分析摘要
  - 持仓变化提醒
  - 风险预警
- **输出**: `reports/daily-market-YYYYMMDD.md`

### 3. 代码验证 (每天)
- **时间**: 22:00 UTC+8
- **内容**:
  - TypeScript编译检查
  - 数据文件完整性
  - Git提交

---

## 二、触发式收集

### 技术指标查询 (实时)
- **来源**: 飞书用户消息
- **触发**: 用户发送 "指标 AAPL" 等
- **处理**: 技术分析 + 卡片回复

### 市场扫描 (开盘期间)
- **频率**: 每5-15分钟
- **内容**:
  - Bollinger挤压检测
  - 涨跌幅排名
  - RSI超卖信号
- **触发条件**: 得分≥70分 → 推送

---

## 三、信息源

### 外部
| 类型 | 来源 | 频率 |
|------|------|------|
| GitHub Trending | github.com/trading | 每周 |
| 技术指标 | TradingView MCP | 实时 |
| 行情数据 | A股/港股/美股API | 实时 |
| 新闻 | 自有新闻采集 | 每天 |

### 内部
| 类型 | 来源 |
|------|------|
| 持仓数据 | PostgreSQL |
| 信号历史 | PostgreSQL |
| 回测结果 | reports/ |

---

## 四、收集格式

### 报告命名
```
reports/{类型}-{日期}.md
```

### 类型前缀
- `竞品分析` - 竞品功能
- `产品优先级` - 产品规划
- `开发进度` - 开发日报
- `daily-market` - 市场日报
- `信息收集` - 临时收集

---

## 五、注意事项

1. ⚠️ 每天必须检查运营状态 (新闻+实盘+日报)
2. 📊 短线信号≥70分立即推送飞书
3. 📝 定期更新 MEMORY.md
4. 🔄 每周收集GitHub Trending
