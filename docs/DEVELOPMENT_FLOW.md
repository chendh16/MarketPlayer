# 开发流程规范

## 原则

1. **本地开发为主** - 代码在本地完成开发和测试
2. **代码检查必须** - 每次提交前必须通过代码检查
3. **GitHub按需** - 仅在功能完成或需要同步时提交

---

## 开发流程

### 1. 本地开发
```
编写代码 → 本地测试 → 代码检查 → 提交GitHub
```

### 2. 代码检查清单

每次提交前必须检查：

- [ ] `npm run build` 编译通过
- [ ] 无新增 TypeScript 错误
- [ ] 无语法错误
- [ ] 导入路径正确

### 3. 检查命令

```bash
# 编译检查
npm run build

# 快速检查
npx tsc --noEmit
```

### 4. GitHub 提交时机

- 功能模块完成
- 需要部署/同步
- 每日结束前

---

## 注意事项

- 禁止未检查直接提交
- 优先本地测试通过再提交
- 保持提交信息清晰
